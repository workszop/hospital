How it works

Index · Embed · Retrieve · Prompt · Answer

1

Document Indexing

Medical literature and clinical guidelines are embedded using a semantic vectorizer and stored in a vector database.

2

Query Understanding

The clinician's question is converted into a semantic vector that captures clinical meaning, not just keywords.

3

Evidence Retrieval

Vector similarity search finds the most contextually relevant medical documents from the knowledge base.

4

Prompt Building

Retrieved evidence is assembled into a structured prompt with system instructions, context documents, and the original query to guide accurate generation.

5

Response Generation

AI processes the constructed prompt, combining retrieved evidence with the clinical query to generate an evidence-based, specific response.

Note: If the question is outside the system's knowledge base (e.g., a non-medical question), the system will indicate that it has no relevant documents to provide an answer.
